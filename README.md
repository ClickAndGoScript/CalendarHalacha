# פרוייקט תוסף לאוצריא

תוסף לתוכנת אוצריא, בנוי באמצעות Otzaria Plugin SDK.

## סקירה כללית

תוסף זה מרחיב את יכולות אוצריא באמצעות טכנולוגיות רשת (HTML, CSS, JavaScript/TypeScript).

## פיתוח

### דרישות מוקדמות

- תוכנת אוצריא רצה במצב debug
- Node.js ו-npm (לתלויות פיתוח)

### התחלת עבודה

1. התקנת תלויות:
   ```bash
   npm install
   ```

2. טעינת התוסף באוצריא:
   - פתחו את אוצריא במצב debug (`flutter run`)
   - נווטו לפאנל התוספים
   - לחצו על אייקון התיקייה בסרגל העליון
   - בחרו את תיקיית השורש של הפרוייקט

3. התוסף יופיע עם תג `DEV` ויתמוך ב-hot reload

### תכונות פיתוח

- **Hot Reload**: שינויים מזוהים אוטומטית וה-WebView מתרענן
- **מסך שגיאות**: שינויים לא תקינים ב-manifest מציגים מסך שגיאה ייעודי
- **הגדרות מתקדמות**: גישה לאפשרויות ייעודיות לפיתוח דרך אייקון ההגדרות

### תלויות אופליין

כל התלויות (Material Design 3, פונטים וכו') מותקנות מקומית בתיקיית `node_modules` ומועתקות אוטומטית לתיקיית `src/lib` בזמן ההתקנה והבנייה. התוסף פועל באופן מלא אופליין ללא תלות ב-CDN חיצוני.

## ניהול גרסאות

הגרסה מנוהלת בקובץ `version.js` בשורש הפרוייקט. לעדכון הגרסה, פשוט ערוך את הקובץ:

```javascript
module.exports = {
  version: "1.0.0"
};
```

הגרסה תסונכרן אוטומטית ל-`manifest.json` ו-`package.json` בזמן הבנייה.

## בנייה ואריזה להפצה

### אריזה מקומית

**macOS/Linux:**
```bash
npm run package
```

**Windows:**
```bash
npm run package:win
```

פעולה זו יוצרת קובץ `.otzplugin` בשורש הפרוייקט עם כל התלויות בפנים.

### יצירת Release ב-GitHub

1. עדכנו את הגרסה בקובץ `version.js`
2. בצעו commit ו-push לשינויים
3. עברו ל-Actions בריפוזיטורי
4. בחרו "Create Release"
5. לחצו "Run workflow"
6. ה-workflow יקרא את הגרסה מ-`version.js`, יבנה את התוסף ויצור release עם הקובץ המוכן
