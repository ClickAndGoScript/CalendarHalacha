# מדריך הרצה ובדיקה בדפדפן - תוסף לוח שנה עברי

מדריך זה מסביר כיצד להריץ את התוסף בדפדפן רגיל לצורך פיתוח, וכיצד לשלוט במשתנים המדמים את סביבת "אוצריא" (Mocking).

## 📋 דרישות מקדימות
*   **Node.js** מותקן (עבור קימפול ה-TypeScript).
*   **VS Code** עם התוסף **Live Server**.

## 🚀 שלבי הרצה מהירים

1.  **קימפול הקוד:** פתחו טרמינל בתיקיית הפרויקט והריצו:
    ```bash
    npm run watch
    ```
    *הערה: פקודה זו תהפוך את קובץ ה-`main.ts` ל-`main.js` ותעדכן אותו בכל שמירה.*

2.  **הרצת השרת:**
    *   פתחו את הקובץ `src/index.html`.
    *   לחצו לחיצה ימנית ובחרו **Open with Live Server**.

## 🛠️ הגדרת המשתנים המדומים (Mocking)

התוסף פועל בדפדפן בזכות סקריפט ב-`src/index.html` שמזהה היעדר סביבת אוצריא ומזריק אובייקט `Otzaria` חלופי. ניתן לערוך את המשתנים הללו כדי לבדוק תרחישים שונים:

### 1. נתוני עליית התוסף (Boot Data)
בתוך `src/index.html`, חפשו את הפונקציה `cb({ ... })` בתוך ה-`plugin.boot`. שם תוכלו לשנות:

*   **Theme (עיצוב):** שינוי צבעי ה-`primary`, `surface` וכו', כדי לראות איך הלוח מגיב לצבעים שונים של האפליקציה.
*   **Platform (פלטפורמה):** שנו את `app.platform` ל-`android` או `ios` כדי לבדוק התאמות למובייל.
*   **Permissions (הרשאות):** ניתן להוסיף או להסיר מחרוזות מהמערך `permissions` כדי לבדוק הודעות שגיאה.

### 2. דימוי פונקציות API (Otzaria.call)
האובייקט המדומה כולל פונקציית `call`. ניתן להוסיף לה `switch-case` כדי להחזיר נתונים שונים:

```javascript
call: async (method, payload) => {
  if (method === 'calendar.getJewishDate') {
    return { 
      success: true, 
      data: { day: 15, month: 1, year: 5785, monthName: 'ניסן' } 
    };
  }
  // ניתן להוסיף כאן דימוי לחיפוש, זמני היום, וכו'
  return { success: true, data: {} };
}
```

### 3. שינוי מצב עיצוב (Dark/Light Mode)
בסקריפט ה-Mock הוספנו פונקציה גלובלית בשם `toggleTheme`. ניתן להפעיל אותה ישירות מה-Console של הדפדפן (F12) כדי לבדוק מעבר דינמי בין מצבים:
```javascript
toggleTheme('dark');  // מעבר למצב כהה
toggleTheme('light'); // מעבר למצב בהיר
```

## ⚠️ פתרון בעיות (Troubleshooting)

### 1. הקוד לא מתעדכן בדפדפן
*   וודאו שהטרמינל מריץ את `npm run watch`.
*   בדקו אם יש שגיאות קימפול בטרמינל (למשל חוסר ב-Type definitions).
*   וודאו שב-`index.html` אתם קוראים ל-`main.js` (הקובץ המקומפל) ולא ל-`main.ts`.

### 2. שגיאת "Otzaria is not defined"
*   שגיאה זו תקרה רק אם הסקריפט של ה-Mock ב-`index.html` נמחק או הועבר למיקום לא נכון. הוא חייב להופיע **לפני** טעינת ה-`main.js`.
